    Create `update/user-management-form.service.ts` with a `UserManagementFormService` that:
    - creates the form group
    - maps `IUser` into form values
    - returns the current `IUser` payload including `authorities`
    - resets the form for both create and edit flows

    The form must include these controls and validator limits from `AdminUserDTO`:
    - `login`: required, min 1, max 50
    - `firstName`: max 50
    - `lastName`: max 50
    - `email`: required, email, min 5, max 254
    - `activated`: boolean
    - `langKey`: required, min 2, max 10
    - `authorities`: string[]

    Create `update/user-management-update.component.ts` and `.html`.
    The component must:
    - export `UserManagementUpdateComponent`
    - use `ChangeDetectionStrategy.OnPush`
    - read the resolved user from route data
    - load all authorities with `UserManagementService.authorities()`
    - build editable role rows with `buildAuthorityRows(..., false)`
    - support both create and edit in one component
    - call `service.create(payload)` on create and `service.update(payload)` on edit
    - keep `isSaving` state and use `finalize(() => this.isSaving.set(false))`
    - route save success to `/admin/users/${saved.login}/view`
    - route cancel to the stored list context when creating and back to detail when editing
    - use `handleHttpError(...)` for unexpected failures and keep inline validation messages under the owning fields when local validators fail

    The template must:
    - use one split-page layout matching the detail route
    - keep the user form card on the left and the role card on the right
    - use PrimeNG form controls plus inline validation text
    - use a dense `p-table` plus `p-checkbox` rows for authorities
    - show role code plus translated label or description in each row
    - show `Create User` on create and `Save User` on edit
    - show `Discard Changes` as the secondary action

    Create `update/user-management-update.component.spec.ts` covering:
    - create flow calling `service.create`
    - edit flow calling `service.update`
    - authority checkbox selection changing the saved `authorities` payload
    - save success navigating to `:login/view`
    - save failure leaving the form visible and exercising the error path

    Update `src/i18n/en/user-management.json` and `src/i18n/vi/user-management.json` with the keys this plan needs:
    - `form.sectionTitle`
    - `form.rolesTitle`
    - `form.save`

    - `form.discard`
    - `form.validation.loginRequired`
    - `form.validation.emailRequired`
    - `form.validation.emailInvalid`
    - `form.validation.maxLength`
    - `form.validation.langKeyRequired`
    - `feedback.createFailed`
    - `feedback.updateFailed`
    - `feedback.saved`
