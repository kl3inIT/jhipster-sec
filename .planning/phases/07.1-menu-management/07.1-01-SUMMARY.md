## Accomplishments

### Task 1: DTOs and Repository Extension

Created four DTO classes following the SecPermissionDTO pattern with Jakarta validation:

- `SecMenuDefinitionDTO` — full menu node shape with `@NotBlank` on menuId/appName/menuName/label and `@NotNull` on ordering
- `SecMenuPermissionDTO` — role+menuId permission shape with `@Pattern(regexp = "ALLOW|DENY")` on effect
- `SyncNodeDTO` — per-node payload for sync requests
- `SyncResultDTO` — immutable seeded/skipped response

Extended `SecMenuPermissionRepository` with `findAllByRole(String role)` and `deleteByAppNameAndMenuId(String appName, String menuId)`.

### Task 2: Admin REST Controllers and Integration Tests

Created two admin controllers following the SecRoleAdminResource hand-mapping pattern:

**SecMenuDefinitionAdminResource** at `/api/admin/sec/menu-definitions`:
- POST (create with duplicate-check), GET (list by appName), GET/{id}, PUT/{id} (mutable fields only), DELETE/{id} (cascade), POST/sync

**AdminMenuPermissionResource** at `/api/admin/sec/menu-permissions`:
- POST (with role existence check, defaults effect to ALLOW), GET (filtered by role+appName), DELETE/{id}

Integration tests (`SecMenuDefinitionAdminResourceIT` + `AdminMenuPermissionResourceIT`) cover all endpoints including cascade delete, sync idempotency, role validation, and 403 for non-admin.
