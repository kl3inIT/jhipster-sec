# Phase 07.1 Plan 03: Menu Access Tab Summary

**Permission matrix page gains a second "Menu Access" tab with p-tree checkbox grants that flush creates/deletes to the backend alongside entity permission changes.**

- **Duration:** ~5 min
- **Started:** 2026-03-25T09:51:00Z

## Accomplishments

### Task 1: Admin Menu Permission Model and Service

Created the data layer connecting the frontend to the Plan 01 backend endpoint:

- `ISecMenuPermissionAdmin` interface with `id`, `role`, `appName`, `menuId`, `effect` fields
- `AdminMenuPermissionService` following `SecPermissionService` pattern:
  - `query(role, appName)` — GET with HttpParams returning plain body `ISecMenuPermissionAdmin[]`
  - `create(permission)` — POST with `observe: 'response'`
  - `delete(id)` — DELETE `/{id}` with `observe: 'response'`
- Uses `ApplicationConfigService.getEndpointFor('api/admin/sec/menu-permissions')`

### Task 2: Permission Matrix Tab Extension

Rewrote permission-matrix component to support two tabs:

**TypeScript changes:**
- Removed `standalone: true` and `SplitterModule`; added `TabsModule`, `TreeModule`, `TranslateModule`
- New state: `menuTreeNodes`, `menuTreeSelection`, `menuGranted`, `menuPendingChanges`, `menuLoading`, `activeTabValue`
- `hasPendingChanges` getter checks both `pendingChanges.size > 0 || menuPendingChanges.size > 0`
- `loadMenuPermissions()` called from `ngOnInit()` — fetches granted permissions, populates `menuGranted`, calls `buildMenuTree()`
- `buildMenuTree()` maps `APP_NAVIGATION_TREE` sections and leaves to `TreeNode[]`; sets initial selection from `menuGranted`
- `onMenuNodeSelect/Unselect` track `menuPendingChanges` correctly (reverts if toggling back to persisted state)
- `discardChanges()` clears `menuPendingChanges` and calls `buildMenuTree()` to restore selection
- `flushChanges()` builds `menuResults$` from `menuPendingChanges` and combines into single `forkJoin([...operations, ...menuResults$])`; discriminates results using `'menuId' in result` type guard

**HTML changes:**
- Replaced `p-splitter` with `p-tabs` / `p-tablist` / `p-tabpanels` / `p-tabpanel` layout
- Entity permissions content (table + attribute table) moved into tab 0 panel
- Menu Access tab (tab 1) contains `p-tree` with `selectionMode="checkbox"`, `[propagateSelectionUp]="false"`, `[propagateSelectionDown]="false"`
- Action bar (Save, Discard, pending count) remains above tabs to control both
- Pending changes span updated to `aria-live="polite"` showing combined count

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] PrimeNG v20+ selector name differs from plan spec**
- **Found during:** Task 2 first build
- **Issue:** Plan specified `p-tab-panel` and `p-tab-list` / `p-tab-panels` — these are not recognized selectors in PrimeNG v20+. The actual selectors are `p-tabpanel`, `p-tablist`, `p-tabpanels`
- **Fix:** Used the correct PrimeNG v20+ selector names; build confirmed correct
- **Files modified:** `permission-matrix.component.html`
- **Commit:** 4de7ef4

**2. [Rule 1 - Bug] p-tabs valueChange emits string | number | undefined**
- **Found during:** Task 2 second build
- **Issue:** Strict TypeScript: `(valueChange)="activeTabValue = $event"` failed because `$event` type is `string | number | undefined`, not `string`
- **Fix:** `(valueChange)="activeTabValue = $event?.toString() ?? '0'"` to safely coerce to string
- **Files modified:** `permission-matrix.component.html`
- **Commit:** 4de7ef4
