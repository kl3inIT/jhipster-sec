  <action>
    Implement request-time authority refresh for JWT-authenticated requests.

    Guardrails:
    - keep JWT as the identity token
    - stop treating the JWT `auth` claim as the final source of authorities
    - load current authorities from persistent state on each request or during principal reconstruction
    - introduce an `AcceptsGrantedAuthorities`-style seam or equivalent so the current principal can receive refreshed authorities
    - ensure downstream consumers such as `MergedSecurityContextBridge` read the refreshed authorities from the current request authentication

    Do not introduce server-side session state to solve this.
  </action>

    <automated>./gradlew integrationTest --tests "com.vn.core.web.rest.AccountResourceIT" --tests "com.vn.core.web.rest.MenuPermissionResourceIT" --tests "com.vn.core.web.rest.SecuredEntityCapabilityResourceIT"</automated>
  </verify>
