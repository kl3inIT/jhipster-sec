# Phase 08.3 Plan 05: Row-Policy Removal Summary

**Row policy is now removed from the product end to end: schema, runtime, admin APIs, frontend UI, translations, and tests no longer depend on it.**

## Accomplishments

- Deleted the backend row-policy model, repository, provider, specification builder, admin resource, DTO or mapper, and their dedicated tests.
- Added the row-policy drop migration and removed the old create migration from the active Liquibase chains, while normalizing remaining role and target typing to the supported values.
- Reworked the secured-entity and admin-role tests to prove the surviving security model without row-policy ownership assumptions.
- Removed the row-policy frontend route, navigation node, page implementation, notification mappings, and translation keys, and simplified the role UI to the remaining `RESOURCE` type.
