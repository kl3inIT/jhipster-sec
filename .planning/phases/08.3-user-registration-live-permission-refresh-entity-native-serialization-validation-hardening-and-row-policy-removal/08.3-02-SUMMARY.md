# Phase 08.3 Plan 02: Live Authority Refresh Summary

**Authority changes now take effect without logout or login: JWT requests rebuild current authorities from database state, and the frontend no longer treats permission-dependent caches as session-static.**

## Accomplishments

- Added `AcceptsGrantedAuthorities` and request-time JWT conversion so authenticated requests carry refreshed DB-backed authorities rather than only the login-time claim snapshot.
- Wired the custom JWT authentication converter into the active resource-server chain and updated the merged security bridge to read authorities from the refreshed principal path.
- Kept frontend permission state aligned by clearing cached menu grants and secured-entity capabilities when authentication-state identity changes.
- Added same-token regression proof to `AccountResourceIT`, `MenuPermissionResourceIT`, and `SecuredEntityCapabilityResourceIT`, then kept the menu-permission service unit proof green.

## Notes

- The backend keeps JWT statelessness intact; no server-side session state was introduced.
