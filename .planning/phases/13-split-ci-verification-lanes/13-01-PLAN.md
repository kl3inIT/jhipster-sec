  <action>
  Create `.github/workflows/backend.yml` with this exact structure:

  ```yaml
  name: Backend CI

  on:
    push:
      branches: [main]
    pull_request:
      branches: [main]

  concurrency:
    group: backend-${{ github.ref }}
    cancel-in-progress: true

  jobs:
    backend:
      name: Backend Tests
      runs-on: ubuntu-latest
      timeout-minutes: 30
      steps:
        - uses: actions/checkout@v4

        - name: Set up JDK 25
          uses: actions/setup-java@v4
          with:
            distribution: temurin
            java-version: 25

        - name: Set up Node.js
          uses: actions/setup-node@v4
          with:
            node-version: 24
            cache: npm

        - name: Set up Gradle
          uses: gradle/actions/setup-gradle@v4

        - name: Install root npm dependencies
          run: npm ci

        - name: Run backend tests
          run: npm run ci:backend:test
  ```

  Key points per decisions:
  - Per D-01: GitHub Actions workflow in `.github/workflows/`
  - Per D-03: This is the backend lane covering Gradle compile + unit tests + integration tests + code quality
  - Per D-04: Calls `npm run ci:backend:test` which chains `backend:info`, `backend:doc:test`, `backend:nohttp:test`, `backend:unit:test`
  - Per D-07: No path-based filtering on any lane
  - Per D-08: Triggers on push to main and pull_request targeting main
  - Concurrency group cancels in-progress runs on same branch to save resources
  - `setup-gradle@v4` handles Gradle wrapper caching automatically (Claude discretion: this is the recommended approach over manual actions/cache)
  - `setup-node@v4` with `cache: npm` handles node_modules caching
  - `timeout-minutes: 30` prevents runaway jobs
  - Root `npm ci` is required because `ci:backend:test` is defined in root package.json
  - Testcontainers (used by integration tests) works on ubuntu-latest runners which provide Docker
  </action>
