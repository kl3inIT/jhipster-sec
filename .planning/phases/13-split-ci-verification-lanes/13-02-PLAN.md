  Create `.github/workflows/prod-validation.yml` with this exact structure:

  ```yaml
  name: Production Validation

  on:
    workflow_dispatch:

  jobs:
    prod-validation:
      name: Production-Like Stack Validation
      runs-on: ubuntu-latest
      timeout-minutes: 20
      steps:
        - uses: actions/checkout@v4

        - name: Set up JDK 25
          uses: actions/setup-java@v4
          with:
            distribution: temurin
            java-version: 25

        - name: Set up Node.js
          uses: actions/setup-node@v4
          with:
            node-version: 24
            cache: npm

        - name: Set up Gradle
          uses: gradle/actions/setup-gradle@v4

        - name: Install root dependencies
          run: npm ci

        - name: Build Jib Docker image
          run: ./gradlew -Pprod jibDockerBuild

        - name: Start production-like stack
          run: docker compose -f src/main/docker/app.yml up -d

        - name: Wait for stack readiness
          run: |
            echo "Waiting for production-like stack to become ready..."
            timeout 120 bash -c 'until curl -sf http://localhost:8080/management/health/readiness; do sleep 2; done'
            echo "Stack is ready."

        - name: Run stack smoke check
          run: npm run phase12:stack:smoke

        - name: Run production-like regression probe
          run: npm run phase12:backend:prodlike

        - name: Teardown stack
          if: always()
          run: docker compose -f src/main/docker/app.yml down -v
  ```

  Key points per decisions:
  - Per D-03: This is the production-validation lane
  - Per D-06: Trigger is `workflow_dispatch` ONLY. No push, no pull_request events. This keeps PR feedback fast while allowing on-demand milestone validation.
  - The workflow first builds the Jib Docker image (`jhipster-sec:latest`) with `./gradlew -Pprod jibDockerBuild`, then starts the full-stack compose from `src/main/docker/app.yml`
  - Waits for readiness by polling `http://localhost:8080/management/health/readiness` with a 120-second timeout (the compose stack includes PostgreSQL and Mailpit health-gated dependencies from Phase 12)
  - Runs `phase12:stack:smoke` which calls `scripts/phase12-stack-smoke.sh` u2014 deterministic smoke check against the live Compose stack
  - Runs `phase12:backend:prodlike` which calls `scripts/phase12-prodlike-regression.mjs` u2014 live-stack regression probe
  - Tears down with `docker compose -f src/main/docker/app.yml down -v` on `always()` to clean up even on failure
  - JDK 25, Gradle, and Node are needed because the Jib build runs through Gradle and npm scripts reference Node
