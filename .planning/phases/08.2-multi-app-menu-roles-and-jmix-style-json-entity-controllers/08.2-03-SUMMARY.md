# Phase 08.2 Plan 03: Raw JSON Secured Query Boundary Summary

**Limited secured JSON query filters now reach `SecureDataManager.loadByQuery`, and protected entity controllers use explicit raw JSON carriers for create, update, and query flows.**

## Performance

- **Duration:** 23 min
- **Started:** 2026-03-27T09:01:00Z
- **Completed:** 2026-03-27T09:23:50Z
- **Tasks:** 2
- **Files modified:** 12

## Accomplishments
- Added a focused `SecureQuerySpecificationFactory` and wired blank-JPQL query parameters into the secured row-filtered load path.
- Moved organization, department, and employee resources away from `Map<String, Object>` request signatures to explicit raw JSON request handling plus `/query` endpoints.
- Extended secured entity enforcement coverage with an explicit `/api/organizations/query` case and kept the brownfield secured CRUD behavior green.

## Task Commits

Each task was committed atomically:

1. **Task 1: Add the limited JSON query contract behind `SecureDataManager.loadByQuery(...)`** - `70d09ac` (feat)
2. **Task 2: Expose raw JSON create/update and explicit `/query` endpoints on protected entities** - `a5cd813` (feat)

## Files Created/Modified
- `src/main/java/com/vn/core/security/data/SecureQuerySpecificationFactory.java` - Builds simple equality specifications from top-level JSON filter maps.
- `src/main/java/com/vn/core/web/rest/vm/SecuredEntityQueryVM.java` - Defines the explicit secured query request body contract.
- `src/main/java/com/vn/core/security/data/SecureDataManagerImpl.java` - Combines row-level specs with JSON filter specs when JPQL is blank.
- `src/main/java/com/vn/core/service/OrganizationService.java` - Adapts SecureDataManager results to JSON trees and exposes query support.
- `src/main/java/com/vn/core/service/DepartmentService.java` - Mirrors the raw JSON and query adaptation for departments.
- `src/main/java/com/vn/core/service/EmployeeService.java` - Mirrors the raw JSON and query adaptation for employees.
- `src/main/java/com/vn/core/web/rest/OrganizationResource.java` - Parses raw JSON request bodies and returns explicit secured JSON query responses.
- `src/main/java/com/vn/core/web/rest/DepartmentResource.java` - Adds the department raw JSON and `/query` boundary.
- `src/main/java/com/vn/core/web/rest/EmployeeResource.java` - Adds the employee raw JSON and `/query` boundary.
- `src/test/java/com/vn/core/security/data/SecureDataManagerImplTest.java` - Proves blank-JPQL filtering and row-spec composition.
- `src/test/java/com/vn/core/web/rest/SecuredEntityEnforcementIT.java` - Verifies `/api/organizations/query` still enforces row and attribute security.

## Decisions Made
- Kept the secured query contract intentionally narrow: top-level equality filters only, no arbitrary JPQL support.
- Left `SecuredEntityQueryVM` as the controller contract and translated it before the service layer to preserve the existing architecture rules.
- Used explicit string-based HTTP JSON serialization in the controllers because direct `JsonNode` HTTP serialization was incorrect under the current runtime stack.
