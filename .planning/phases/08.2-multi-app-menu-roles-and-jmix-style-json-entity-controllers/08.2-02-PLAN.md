  <action>
    In `admin-menu-permission.service.ts`, change the service contract to:
    - `query(role: string, appName?: string): Observable<ISecMenuPermissionAdmin[]>`
    - only append the `appName` query param when a non-empty `appName` argument is provided
    - keep `create(...)` and `delete(...)` unchanged

    In `permission-matrix.component.ts`:
    - inject `SecMenuDefinitionService`
    - add explicit app state for the menu tab: `availableMenuApps`, `selectedMenuApp`, and `menuDefinitions`
    - load all menu grants for the role by calling `adminMenuPermissionService.query(this.roleName)` without an app filter
    - derive `availableMenuApps` from the returned permission rows, and always include `'jhipster-security-platform'` as a fallback default when the role has no existing rows
    - replace plain `menuId`-only keying with a composite key format such as `${appName}::${menuId}` everywhere menu grants and pending changes are tracked
    - add `loadMenuDefinitions(appName: string)` that calls `SecMenuDefinitionService.query(appName)` and stores the returned definition rows for tree building
    - build the menu tree from backend `SecMenuDefinition` rows instead of importing `APP_NAVIGATION_TREE`
    - when the selected app changes, rebuild the menu tree from the selected app's definitions while preserving pending changes for other apps in memory
    - make save operations send `appName: selectedMenuApp` for newly created rows and delete the exact permission id for the selected-app grant being unchecked

    Keep entity permission behavior unchanged.
