## Jmix-aligned

- Protected entity create, update, and query endpoints use raw JSON request carriers at the web boundary and serialize secured responses explicitly.
- Secured entity reads expose an explicit `/query` endpoint that reaches `SecureDataManager.loadByQuery` through the limited JSON filter contract shipped in this phase.
- Partial updates leave omitted fields untouched because the explicit `PATCH` path forwards only provided JSON keys into the existing secure merge and save flow.

## Intentionally project-specific

- The secured query body supports the repository's limited JSON filter contract instead of arbitrary JPQL text.
- Protected entities now expose explicit `PATCH` endpoints in addition to the existing update semantics so HTTP partial-update intent is visible at the boundary.
- Fetch plans remain YAML/code-defined only; this phase does not introduce database-backed fetch-plan metadata.
