| 3 | Protected entity controllers use a Jmix-style raw JSON boundary for create, update, and query operations instead of exposing `Map<String, Object>` request signatures. | VERIFIED | `OrganizationResource`, `DepartmentResource`, and `EmployeeResource` accept raw JSON request bodies, parse them into `JsonNode`, serialize explicit JSON responses, and delegate to `JsonNode`-based services instead of `Map<String, Object>` controller signatures. |
| 4 | `loadByQuery` is restored and verified end to end as a first-class secured capability. | VERIFIED | All three protected entity resources expose `/query`. Services build `SecuredLoadQuery`, and `SecureDataManagerImpl.loadByQuery(...)` combines row-level restrictions with `SecureQuerySpecificationFactory` filters while still rejecting nonblank JPQL. `SecuredEntityEnforcementIT` covers `/api/organizations/query`, and `SecureDataManagerImplTest` keeps filter composition and fail-closed JPQL behavior green. |
| 5 | `PATCH` accepts raw JSON request bodies, updates only provided fields, preserves omitted fields, and continues to enforce CRUD, row-level, attribute-level, and fetch-plan security on protected entities. | VERIFIED | Organization, department, and employee resources expose explicit `@PatchMapping` handlers. Their services forward only provided JSON keys into `secureDataManager.save(...)`, which performs row-constrained target lookup and calls `SecureMergeService.mergeForUpdate(...)`. `SecuredEntityEnforcementIT` proves omitted `code` remains unchanged and denied `budget` edits return `403 Forbidden`. |
| 6 | Brownfield auth/account/admin/mail flows and backend-driven navigation do not regress, and intentional contract changes are documented before handoff. | VERIFIED | `08.2-ALIGNMENT.md` records Jmix-aligned vs project-specific behavior. A live Java 25 verification run of `AccountResourceIT`, `UserResourceIT`, `SecuredEntityEnforcementIT`, `MenuPermissionResourceIT`, `AdminMenuPermissionResourceIT`, and `SecMenuDefinitionAdminResourceIT` passed. The Phase 08.2 code diff does not modify auth, account, or mail runtime classes. |
