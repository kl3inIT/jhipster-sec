    **1. `RolePermissionServiceDbImpl.java`** — `@Service` implementing `RolePermissionService`:
    - Constructor-inject `SecPermissionRepository` and `MergedSecurityService`
    - `isEntityOpPermitted(Class<?> entityClass, EntityOp op)`:
      - Get authority names from `mergedSecurityService.getCurrentUserAuthorityNames()`
      - If empty, return `false`
      - Normalize target: `entityClass.getSimpleName().toUpperCase(Locale.ROOT)`
      - Call `hasPermission(authorities, TargetType.ENTITY, target, op.name())`
    - `hasPermission(Collection<String> authorityNames, TargetType targetType, String target, String action)`:
      - Call `secPermissionRepository.findByRolesAndTarget(authorityNames, targetType, target, action)`
      - **DENY-wins per D-07:** `boolean hasDeny = perms.stream().anyMatch(p -> "DENY".equals(p.getEffect()));`
      - If `hasDeny`, return `false`
      - Return `perms.stream().anyMatch(p -> "ALLOW".equals(p.getEffect()))`
      - Note: `SecPermission.effect` is a String, not an enum — compare with `"DENY"` and `"ALLOW"` strings

    **2. `EntityPermissionEvaluatorImpl.java`** — `@Component` implementing `EntityPermissionEvaluator`:

    - `isPermitted(Class<?> entityClass, EntityOp op)`:

    **3. `AttributePermissionEvaluatorImpl.java`** — `@Component` implementing `AttributePermissionEvaluator`:
    - Constructor-inject `RolePermissionService` and `MergedSecurityService`
    - `canView(Class<?> entityClass, String attribute)`:
      - Normalize target: `entityClass.getSimpleName().toUpperCase(Locale.ROOT) + "." + attribute.toUpperCase(Locale.ROOT)`
      - Call `rolePermissionService.hasPermission(mergedSecurityService.getCurrentUserAuthorityNames(), TargetType.ATTRIBUTE, target, "VIEW")`
      - If no permission records exist for this attribute, default to `true` (absence of rules = visible)
    - `canEdit(Class<?> entityClass, String attribute)`:
      - Same normalization
      - Call with `"EDIT"` action
      - If no permission records exist for this attribute, default to `true`
    - **Important detail on defaults:** The `hasPermission` method returns `false` when no matching permissions exist. For attribute-level permissions, the desired default is permissive (no rules = allowed). So `AttributePermissionEvaluatorImpl` must distinguish "no rules at all" from "has DENY rule". Add a helper or call `secPermissionRepository.findByRolesAndTarget` directly to check: if the result list is empty, return `true`; otherwise delegate to `hasPermission`.
    - **Resolution:** Constructor-inject `SecPermissionRepository` alongside `RolePermissionService`. In `canView`/`canEdit`, first call `secPermissionRepository.findByRolesAndTarget(authorities, TargetType.ATTRIBUTE, target, actionStr)`. If the returned list is empty, return `true`. Otherwise, check for DENY-wins in the list directly: `hasDeny -> false`, else `hasAllow -> true`, else `false`.

    **4. `RowLevelPolicyProviderDbImpl.java`** — `@Service` implementing `RowLevelPolicyProvider`:
    - Constructor-inject `SecRowPolicyRepository` and `MergedSecurityService`
    - `getPolicies(String entityName, EntityOp operation)`:
      - Call `secRowPolicyRepository.findByEntityNameAndOperation(entityName, operation.name())`
      - For each `SecRowPolicy`, convert to a `RowPolicyDefinition`:
        - If `policyType` equals `"SPECIFICATION"`: parse the expression into a `Specification` using the simple DSL
        - If `policyType` equals `"JPQL"`: parse the WHERE fragment with token substitution
        - **Fail-closed per D-13:** If the expression cannot be parsed, throw `new AccessDeniedException("Unsafe row policy: " + policy.getCode() + " — cannot parse expression")`
      - **Supported SPECIFICATION expressions per D-11/D-12:**
        - Format: `field = CURRENT_USER_LOGIN` where `CURRENT_USER_LOGIN` resolves to `mergedSecurityService.getCurrentUserLogin().orElseThrow()`
        - The Specification lambda: `(root, query, cb) -> cb.equal(root.get(field), resolvedValue)`
      - **Supported JPQL WHERE fragment per D-11/D-12:**
        - The expression is a WHERE clause fragment like `e.createdBy = :currentUserLogin`
        - Built-in tokens: `{CURRENT_USER_LOGIN}` replaced with the current user login
        - Create a `RowPolicyDefinition` whose `getSpecification()` constructs a JPA Criteria predicate from the parsed fragment
        - For simple equality patterns (`field = :currentUserLogin`), extract field name and resolve token, then build `cb.equal(root.get(field), resolvedValue)`
        - For unsupported/complex patterns: throw `AccessDeniedException` per D-13
      - **D-14:** Skip any policy with `policyType` equals `"JAVA"` — throw `AccessDeniedException("JAVA row policies not supported in Phase 3")`

    **5. `RowLevelSpecificationBuilder.java`** — `@Component`:
    - Constructor-inject `RowLevelPolicyProvider`
    - `public <T> Specification<T> build(Class<T> entityClass, EntityOp op)`:
      - Call `provider.getPolicies(entityClass.getSimpleName(), op)`
      - If no policies: return `Specification.where(null)` (no constraint)
      - AND-compose all returned `RowPolicyDefinition.getSpecification()` instances:
        ```java
        Specification<T> combined = Specification.where(null);
        for (RowPolicyDefinition def : policies) {
            combined = combined.and(def.getSpecification());
        }
        return combined;
        ```
      - This ensures row policies are applied as database-level constraints per D-10
