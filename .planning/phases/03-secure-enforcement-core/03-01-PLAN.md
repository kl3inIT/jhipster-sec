    1. `AccessContext.java` — marker interface in `com.vn.core.security.access`:
       ```java
       public interface AccessContext {}
       ```

    2. `AccessConstraint.java` — generic constraint interface:
       ```java
       public interface AccessConstraint<C extends AccessContext> {
           Class<C> supports();
           void applyTo(C context);
           default int getOrder() { return 0; }
       }
       ```

    3. `AccessManager.java` — pipeline interface:
       ```java
       public interface AccessManager {
           <C extends AccessContext> C applyRegisteredConstraints(C context);
       }
       ```

    4. `AccessManagerImpl.java` — `@Component` implementation. Constructor-inject `List<AccessConstraint<?>>`. The `applyRegisteredConstraints` method:
       - Streams all constraints
       - Filters by `c.supports().isAssignableFrom(context.getClass())`
       - Sorts by `Comparator.comparingInt(AccessConstraint::getOrder)`
       - For each matching constraint, casts `(AccessConstraint<C>) c` and calls `c.applyTo(context)`
       - Returns the context
       - Suppress `"unchecked"` on the cast

    5. `CrudEntityContext.java` — mutable context class:
       - Fields: `Class<?> entityClass`, `EntityOp operation`, `boolean permitted` (default `false` per research — missing constraint grants nothing)
       - Constructor: `CrudEntityContext(Class<?> entityClass, EntityOp operation)`
       - Getters + `setPermitted(boolean)` setter
       - Implements `AccessContext`

    6. `FetchPlanAccessContext.java` — mutable context:
       - Fields: `Class<?> entityClass`, `String fetchPlanCode`, `boolean permitted` (default `true` — catalog allowlist is the gate per research)
       - Constructor, getters, `setPermitted(boolean)`
       - Implements `AccessContext`

    7. `AttributeAccessContext.java` — mutable context:
       - Fields: `Class<?> entityClass`, `String attribute`, `String action` (VIEW or EDIT), `boolean permitted` (default `false`)
       - Constructor, getters, `setPermitted(boolean)`
       - Implements `AccessContext`

    8. `RowLevelAccessContext.java` — mutable context:
       - Fields: `Class<?> entityClass`, `EntityOp operation`, `List<jakarta.persistence.criteria.Predicate> predicates` (mutable, starts empty)
       - Constructor, getters, `addPredicate(Predicate)` method
       - Implements `AccessContext`

    9. `CrudEntityConstraint.java` — `@Component` implementing `AccessConstraint<CrudEntityContext>`:

       - Import `RolePermissionService` from `com.vn.core.security.permission`
