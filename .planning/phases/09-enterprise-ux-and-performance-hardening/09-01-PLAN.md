# Plan 1: Backend Permission Caching

**Phase:** 9 тАФ Enterprise UX And Performance Hardening
**Goal:** Eliminate the N+1 permission query explosion on secured entity list requests by caching authority and permission lookups in a request-scoped Spring bean, reducing ~202 DB queries per 20-row department list to 2.
**Verification:** `./gradlew integrationTest` passes green; a department list request no longer triggers per-row authority or permission queries.

## Context

Every secured entity list request currently triggers authority validation (`authorityRepository.findAllById`) and permission lookup (`secPermissionRepository.findByRolesAndTarget/findByRolesAndTargets`) once per CRUD check and once per attribute per row. For a 20-row department list with 5 attributes, that is 202 DB queries. The `SecuredEntityCapabilityService` already demonstrates the correct pattern: load all permissions once via `findAllByAuthorityNameIn`, build an in-memory `PermissionMatrix`, and answer all checks from it. This plan extracts that pattern into a request-scoped bean and wires it into the three hot-path callers.

Locked decisions honored:
- **D-01**: Snapshot is per-request only; destroyed at request end.
- **D-02**: Request-local reuse only; no cross-request or session caching.
- **D-03**: Reuse the existing `PermissionMatrix` pattern from `SecuredEntityCapabilityService`.
- **D-04**: Integration tests already pass (verified by research); this plan includes a baseline verification task only.

## Tasks

### Task 1: Extract PermissionMatrix to a shared package-level class and create RequestPermissionSnapshot
**File(s):**
- `src/main/java/com/vn/core/security/permission/PermissionMatrix.java` тАФ CREATE
- `src/main/java/com/vn/core/security/permission/RequestPermissionSnapshot.java` тАФ CREATE
- `src/main/java/com/vn/core/service/security/SecuredEntityCapabilityService.java` тАФ MODIFY

**Action:**

1. **Extract `PermissionMatrix`** from the `private static class` inside `SecuredEntityCapabilityService` (lines 100-128) into a new top-level package-accessible class at `src/main/java/com/vn/core/security/permission/PermissionMatrix.java`:
   - Keep the same `allowedKeys` Set implementation and `isEntityPermitted`/`isAttributePermitted` methods.
   - Keep the `EMPTY` static constant.
   - Constructor takes `List<SecPermission>`.
   - Package: `com.vn.core.security.permission`.

2. **Update `SecuredEntityCapabilityService`** to import and use the extracted `PermissionMatrix` instead of its private inner class. Remove the inner class definition. The rest of the service logic stays identical.

3. **Create `RequestPermissionSnapshot`** at `src/main/java/com/vn/core/security/permission/RequestPermissionSnapshot.java`:
   - Annotate with `@Component` and `@Scope(value = "request", proxyMode = ScopedProxyMode.TARGET_CLASS)`.
   - Constructor-inject `AuthorityRepository` and `SecPermissionRepository` directly (NOT `SecurityContextBridge` тАФ this avoids any circular call chain).
   - Private fields: `Collection<String> cachedAuthorities` (initially null), `PermissionMatrix cachedMatrix` (initially null).
   - `public Collection<String> getAuthorities()`: if `cachedAuthorities` is null, resolve JWT authority names from `SecurityContextHolder.getContext().getAuthentication()`, validate against `authorityRepository.findAllById(jwtAuthorities)` to filter to known authorities, and cache; return cached value. This is the same filtering logic already in `MergedSecurityContextBridge.resolveAuthorities()` тАФ copy that logic directly.
   - `public PermissionMatrix getMatrix()`: if `cachedMatrix` is null, call `getAuthorities()`, then `secPermissionRepository.findAllByAuthorityNameIn(authorities)` (or return `PermissionMatrix.EMPTY` if authorities empty), build and cache a `PermissionMatrix`; return cached value.
   - **Graceful degradation**: Add a static helper method `public static boolean isRequestScopeActive()` that returns `RequestContextHolder.getRequestAttributes() != null`. Callers will use this to decide whether to use the snapshot or fall back to direct queries.
   - Note: `SecuredEntityCapabilityService` also uses a `PermissionMatrix` but loads its own independently for the capability-summary endpoint. It is intentionally NOT wired to `RequestPermissionSnapshot` тАФ it already bulk-loads its own matrix once per capability request and the extra wiring would add complexity for no gain.

**Acceptance:**
- `./gradlew classes` compiles without errors.
- `SecuredEntityCapabilityService` still references `PermissionMatrix` (now from `com.vn.core.security.permission` package).
- `RequestPermissionSnapshot` class exists with `@Component` + `@Scope("request")` annotations.

### Task 2: Wire RequestPermissionSnapshot into MergedSecurityContextBridge, RolePermissionServiceDbImpl, and AttributePermissionEvaluatorImpl
**File(s):**
- `src/main/java/com/vn/core/security/bridge/MergedSecurityContextBridge.java` тАФ MODIFY
- `src/main/java/com/vn/core/security/permission/RolePermissionServiceDbImpl.java` тАФ MODIFY
- `src/main/java/com/vn/core/security/permission/AttributePermissionEvaluatorImpl.java` тАФ MODIFY

**Action:**

1. **MergedSecurityContextBridge** тАФ Inject `RequestPermissionSnapshot` (via scoped proxy, so it resolves per-request). In `getCurrentUserAuthorities()`:
   - If `RequestPermissionSnapshot.isRequestScopeActive()`, delegate entirely to `requestPermissionSnapshot.getAuthorities()` and return.
   - Otherwise, fall back to the existing `authorityRepository.findAllById(jwtAuthorities)` filtering logic (keep `resolveAuthorities(auth)` private method unchanged).
   - The snapshot loads authorities directly from `AuthorityRepository` (see Task 1 тАФ no circular call chain exists). The bridge simply delegates to the snapshot's already-cached result on subsequent calls in the same request.
   - Update the constructor to accept `RequestPermissionSnapshot` as an additional parameter alongside the existing `AuthorityRepository`.

2. **RolePermissionServiceDbImpl** тАФ Inject `RequestPermissionSnapshot`. In `isEntityOpPermitted(Class<?>, EntityOp)`:
   - If request scope is active, use `snapshot.getMatrix().isEntityPermitted(target, op.name())` and return the result directly.
   - Otherwise, fall back to the existing `mergedSecurityService.getCurrentUserAuthorityNames()` + `secPermissionRepository.findByRolesAndTarget(...)` logic.
   - The `hasPermission(Collection<String>, TargetType, String, String)` method remains unchanged (it is a general-purpose method that may be called with explicit authority lists).
   - Update the constructor to accept `RequestPermissionSnapshot` as an additional parameter.

3. **AttributePermissionEvaluatorImpl** тАФ Inject `RequestPermissionSnapshot`. In `checkAttributePermission(Class<?>, String, String)`:
   - If request scope is active, use `snapshot.getMatrix().isAttributePermitted(specificTarget, action)` (which already handles wildcard internally) and return the result.
   - Otherwise, fall back to the existing `mergedSecurityService.getCurrentUserAuthorityNames()` + `secPermissionRepository.findByRolesAndTargets(...)` logic.
   - Update the constructor to accept `RequestPermissionSnapshot` as an additional parameter.

**Acceptance:**
- `./gradlew classes` compiles without errors.
- All three classes have `RequestPermissionSnapshot` injected and use it when request scope is active.
- The fallback path preserves the original query logic for non-web contexts.
