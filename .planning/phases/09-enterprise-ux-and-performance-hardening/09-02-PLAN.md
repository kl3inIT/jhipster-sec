# Plan 2: Frontend Entity List Hardening

**Phase:** 9 — Enterprise UX And Performance Hardening
**Goal:** Migrate Department, Employee, and Organization list components to signal-based pagination with OnPush change detection, add skeleton loaders for initial data fetch, and add responsive column hiding at tablet widths.
**Verification:** `cd frontend && npm run build` succeeds; pagination signals update correctly on page changes; skeleton rows appear during initial load; lower-priority columns hide below 1024px viewport width.

## Context

The three entity list components (Department, Employee, Organization) already have working server-side pagination via `p-table [lazy]="true" [paginator]="true"` with backend `Pageable` + `X-Total-Count`. However, `totalItems`, `page`, and `firstRow` are plain mutable fields/getters rather than signals, which will break once `ChangeDetectionStrategy.OnPush` is added. The user-management list component is the reference implementation with `totalItems = signal(0)`, `page = signal(1)`, and `firstRow = computed(...)`.

Additionally, these lists lack skeleton loaders (they show a PrimeNG loading overlay but no content placeholder on first load) and have no responsive column hiding for tablet widths. PrimeNG Skeleton is already installed. `@angular/cdk` is NOT in `package.json`, so responsive detection uses `fromEvent(window, 'resize')` with `debounceTime`.

Locked decisions honored:
- **D-05**: Server-side pagination already works; this plan fixes the signal/state alignment gap.
- **D-06**: Skeleton loaders for initial fetch + responsive column hiding at tablet widths (< 1024px).

**B. Responsive column hiding (all three components):**

1. Add to each component TS file:
```typescript
import { DestroyRef, AfterViewInit } from '@angular/core';
import { fromEvent, debounceTime, startWith } from 'rxjs';
import { takeUntilDestroyed } from '@angular/core/rxjs-interop';
```

2. Add a signal and lifecycle hook:
```typescript
private readonly destroyRef = inject(DestroyRef);
readonly isTablet = signal(typeof window !== 'undefined' ? window.innerWidth < 1024 : false);
```

3. Implement `AfterViewInit` (add to class declaration: `implements OnInit, OnDestroy, AfterViewInit`):
```typescript
ngAfterViewInit(): void {
  fromEvent(window, 'resize')
    .pipe(
      debounceTime(150),
      startWith(null),
      takeUntilDestroyed(this.destroyRef),
    )
    .subscribe(() => {
      this.isTablet.set(window.innerWidth < 1024);
    });
}
```

4. Add responsive computed signals per entity:

**Department:**
```typescript
showIdColumn = computed(() => !this.isTablet());
// Modify existing signals:
showOrganizationColumn = computed(() => !this.isTablet() && this.canViewField('organization'));
showCostCenterColumn = computed(() =>
  !this.isTablet() && this.canViewField('costCenter') && this.departments().some(d => d.costCenter !== undefined),
);
```
Add `showIdColumn` guard to the template's `id` column header and body cells (wrap with `@if (showIdColumn())`).

**Employee:**
```typescript
showIdColumn = computed(() => !this.isTablet());
showDepartmentColumn = computed(() => !this.isTablet());
// Existing showSalaryColumn already conditional; extend:
showSalaryColumn = computed(() => !this.isTablet() && this.employees().some(e => e.salary !== undefined));
```
Add `showIdColumn` and `showDepartmentColumn` guards to template. Wrap the id column header+body and department column header+body with `@if`.

**Organization:**
```typescript
// NEW signals — these do not exist today; add them:
```

The `id` and `ownerLogin` columns are currently rendered unconditionally in the Organization template. After adding these signals, wrap those columns in `@if (showIdColumn())` and `@if (showOwnerLoginColumn())` in the HTML template (both header `<th>` and body `<td>` cells).
