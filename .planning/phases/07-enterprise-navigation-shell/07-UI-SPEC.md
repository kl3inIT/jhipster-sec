## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage | Source |
|-------|-------|-------|--------|
| xs | 4px | Breadcrumb separators, inline icon gaps, dense status spacing | default for Phase 7 |
| sm | 8px | Compact control gaps, topbar grouped controls, submenu icon spacing | aligned to existing shell gaps |
| md | 16px | Default control padding, card internals, row action spacing | aligned to current form and table rhythm |
| lg | 24px | Workspace section padding and header-to-content separation | aligned to current entity page spacing |
| xl | 32px | Desktop shell gutters and main layout breathing room | `frontend/src/assets/layout/_main.scss` plus Phase 7 default |
| 2xl | 48px | Major empty, denied, and error state spacing | default for Phase 7 |

Source: `frontend/src/assets/layout/_menu.scss`, `frontend/src/assets/layout/_main.scss`, `frontend/src/assets/layout/_responsive.scss`, WCAG AA default.

## Navigation Tree Contract

### Stable node id naming

- Root sections use fixed ids: `home`, `entities`, `security`.
- Visible leaves use lowercase dot-delimited ids owned by the frontend: `home.dashboard`, `entities.organization`, `entities.department`, `entities.employee`, `security.users`, `security.roles`, `security.row-policies`.
- Detail, edit, and create routes do not get separate backend-managed menu ids. They inherit the parent leaf id for highlighting and breadcrumb ancestry.

### Ownership and filtering

| Contract | Specification | Source |
|----------|---------------|--------|
| Tree ownership | The frontend owns section ordering, translation keys, icons, router targets, and stable node ids. | `D-01`, `D-05` |
| Backend payload | Backend returns only app-scoped allowed node ids for the current user. It does not return labels, ordering, or rendered tree structure. | `D-02`, `D-03` |
| App scope | Use a single explicit app identifier for this shell: `jhipster-security-platform`. Backend records may store it as `app_name`. | Phase 7 default within `D-03` discretion |
| Section visibility | Hide a root section when all of its child leaves are denied. | `D-04` |
| Admin leaf visibility | Hide denied admin leaves completely. A user must never see a visible-but-denied admin menu leaf. | `D-16` |
| Entity leaf visibility | Show entity leaves when navigation grant allows route entry, even if entity `READ` is false. | `D-14` |
| Active highlighting | Exactly one leaf is active. Deep routes highlight the owning leaf and keep its parent section expanded. | `D-12` plus current `activePath` shell behavior |

### Canonical leaf inventory

| Section | Leaf id | Label source key | Icon | Route |
|---------|---------|------------------|------|-------|
| Home | `home.dashboard` | `global.menu.home` | `pi pi-home` | `/` |
| Entities | `entities.organization` | `global.menu.entities.organization` | `pi pi-building` | `/entities/organization` |
| Entities | `entities.department` | `global.menu.entities.department` | `pi pi-sitemap` | `/entities/department` |
| Entities | `entities.employee` | `layout.menu.entities.employee` | `pi pi-users` | `/entities/employee` |

| Security | `security.row-policies` | `layout.menu.security.rowPolicies` | `pi pi-filter` | `/admin/security/row-policies` |

## Breadcrumb, Route, And Recovery Contract

| Scenario | Contract | Source |
|----------|----------|--------|
| Breadcrumb presence | All non-home admin and entity routes show breadcrumbs above the page title. | `D-12` |
| Breadcrumb structure | Pattern is `Section / Leaf / View state`. Use translated section and leaf labels. View-state labels are `Details`, `Edit`, or `New`. | Phase 7 default consistent with `D-10` to `D-13` |
| Breadcrumb current item | Final crumb uses accent color, 600 weight, and `aria-current="page"`. All earlier crumbs are neutral text links. | Phase 7 default |
| Deep-link denial | Unauthorized deep links redirect to `/accessdenied` before protected component content renders. Never silently reroute to home. | `D-06`, `ROUTE-02` |
| Blocked destination messaging | The denied page names the blocked destination using the translated breadcrumb or leaf label when available, else route title. | `D-07` |
| Safe fallback CTA | Primary recovery action resolves in this order: first allowed sibling leaf in the same section, else first allowed leaf in frontend section order, else `/`. | Phase 7 default within `D-09` discretion |
| Parent section behavior | If a denied leaf belongs to a section with accessible siblings, keep the parent section visible and hide only the denied leaf. | `D-08` |
| List-first routing | Admin and entity sections land on list or index routes. Detail and edit are subordinate drill-in pages, not modal defaults. | `D-10`, `D-13` |
| List-state preservation | Returning from detail or edit to list must restore filters, pagination, sort, and section selection using route query params or stored route state. Hardcoded `/entities/foo` back buttons are not acceptable for touched Phase 7 screens. | `D-11` |
| Lazy boundaries | Keep `admin` and `entities` behind separate lazy route boundaries. Menu filtering must not force eager loading of both sections. | `ROUTE-03`, `frontend/src/app.routes.ts` |

### Entity denied-state rule

- If a user can open an entity leaf from navigation but lacks entity `READ`, the route stays inside the shell and renders a full-page denied state in the workspace body.
- That denied state must not start the list data request.
- Create and edit flows remain stricter: if capability denies `CREATE` or `UPDATE`, redirect to `/accessdenied` before form controls render.

Source: `D-14`, `D-15`, Phase 5 locked routing behavior, current entity route and component capability patterns.

## Accessibility Contract

| Area | Requirement | Source |
|------|-------------|--------|
| Focus | Route changes to denied, empty, and error states must place focus on the page H1. | WCAG AA default for Phase 7 |
| Navigation semantics | Collapsible menu parents expose `aria-expanded`; current route leaf exposes `aria-current="page"` or equivalent active semantics. | WCAG AA default plus current menu structure |
| Hit targets | Icon-only controls in topbar, sidebar collapse, and recovery actions must meet 44x44px minimum touch target. | WCAG AA default |
| Contrast | Accent on dominant or secondary surfaces must meet WCAG AA. Use Prime token colors, not ad hoc lighter greens. | project accessibility requirement plus color contract |
| Motion reduction | Navigation shell must remain usable with reduced motion; submenu animation cannot be required to communicate state. | Phase 7 default |

## Translation Contract

| Requirement | Contract | Source |
|-------------|----------|--------|

| Destination labels | Access-denied and breadcrumb copy must resolve from the same translation source used by the visible menu leaf. | Phase 7 default |
