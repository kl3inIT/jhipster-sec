  <action>
Create `frontend/src/app/layout/navigation/navigation.service.ts` as a root service that:
- calls `ApplicationConfigService.getEndpointFor('api/security/navigation-grants')`
- requests the backend with the query parameter `appName=jhipster-security-platform`
- stores the response contract `{ appName, allowedNodeIds }`
- caches the current-user grant payload in memory and sessionStorage
- clears both caches when `AccountService.getAuthenticationState()` emits a different login
- exposes methods:
  - `query()`
  - `allowedNodeIds()`
  - `visibleTree()`
  - `isNodeVisible(nodeId: string)`
  - `resolveFallbackRoute(sectionId?: string)`
  - `getLeaf(nodeId: string)`

`visibleTree()` must filter leaves based on `allowedNodeIds` and remove any root section whose child list becomes empty.
`resolveFallbackRoute(sectionId?: string)` must prefer:
1. the first allowed leaf in the requested section
2. the first allowed leaf in registry order
3. `/`

Create `frontend/src/app/layout/navigation/navigation.service.spec.ts` covering:
- initial backend fetch with `appName=jhipster-security-platform`
- no second HTTP request after warm-cache reuse
- sessionStorage warm start
- cache reset on auth-state change
- hidden empty sections after filtering
- fallback route selection when a section has no accessible sibling
