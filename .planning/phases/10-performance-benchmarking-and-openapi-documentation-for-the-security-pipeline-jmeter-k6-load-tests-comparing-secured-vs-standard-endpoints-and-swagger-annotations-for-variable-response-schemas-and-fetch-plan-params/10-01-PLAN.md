**1. Create `src/main/java/com/vn/core/web/rest/BenchmarkOrganizationResource.java`:**

A profile-gated controller that serves Organization data WITHOUT the security pipeline, for fair latency comparison against the secured `OrganizationResource`. Per D-07, this is the "baseline" that bypasses `SecureDataManagerImpl`.

```java
package com.vn.core.web.rest;

// Use these imports:
import com.vn.core.domain.Organization;
import com.vn.core.security.data.UnconstrainedDataManager;
import com.vn.core.security.web.SecuredEntityJsonAdapter;
import io.swagger.v3.oas.annotations.Hidden;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springdoc.core.annotations.ParameterObject;
import org.springframework.context.annotation.Profile;
import org.springframework.data.domain.Page;
import org.springframework.data.domain.Pageable;
import org.springframework.http.MediaType;
import org.springframework.http.ResponseEntity;
import org.springframework.security.access.prepost.PreAuthorize;
import org.springframework.transaction.annotation.Transactional;
import org.springframework.web.bind.annotation.*;
```

Key implementation details:
- Annotate class with `@Profile("api-docs")` — only active in dev (per RESEARCH open question 2)
- Annotate class with `@Hidden` from `io.swagger.v3.oas.annotations` — excludes from OpenAPI spec (per RESEARCH pitfall 5)
- Map to `@RequestMapping("/api/benchmark/organizations")`
- Add `@PreAuthorize("isAuthenticated()")` — still requires JWT auth (same as secured endpoint)
- Constructor-inject `UnconstrainedDataManager` and `SecuredEntityJsonAdapter`
- `private static final Logger LOG = LoggerFactory.getLogger(BenchmarkOrganizationResource.class);`
- Use `private static final String LIST_FETCH_PLAN = "organization-list";` and `private static final String DETAIL_FETCH_PLAN = "organization-detail";`

**List endpoint** `@GetMapping("")` with `@Transactional(readOnly = true)`:
- Call `unconstrainedDataManager.loadPage(Organization.class, null, pageable)` — `null` spec means no restriction (safe per Spring Data JPA docs, confirmed in RESEARCH)
- Serialize with `securedEntityJsonAdapter.toJsonArrayString(page.getContent(), LIST_FETCH_PLAN)` — uses the SAME serializer as the secured path but skips CRUD check + permission cache lookup. This makes the benchmark comparison fair: only the security overhead is measured, not serialization differences (per RESEARCH pitfall 3 resolution)
- Return `ResponseEntity.ok().contentType(MediaType.APPLICATION_JSON).body(json)`

**Detail endpoint** `@GetMapping("/{id}")` with `@Transactional(readOnly = true)`:
- Call `unconstrainedDataManager.load(Organization.class, id)` — direct load, no CRUD check
- Serialize with `securedEntityJsonAdapter.toJsonString(organization, DETAIL_FETCH_PLAN)`
- Return 200 with JSON body, or 404 if entity is null (wrap in null check)

**2. Create `load-tests/scripts/auth.js`:**

Shared authentication helper module for k6 scripts.

```javascript
import http from 'k6/http';

const BASE_URL = __ENV.BASE_URL || 'http://localhost:8080';

export function authenticate(username, password) {
  const res = http.post(
    `${BASE_URL}/api/authenticate`,
    JSON.stringify({ username, password, rememberMe: true }),
    { headers: { 'Content-Type': 'application/json' } }
  );
  if (res.status !== 200) {
    throw new Error(`Authentication failed: ${res.status} ${res.body}`);
  }
  return res.json('id_token');
}

export function authHeaders(token) {
  return {
    Authorization: `Bearer ${token}`,
    Accept: 'application/json',
  };
}
```

- `rememberMe: true` to get a long-lived token (per RESEARCH pitfall 4)
- `BASE_URL` configurable via k6 `--env BASE_URL=...` for flexibility

**3. Create `load-tests/scripts/org-list-benchmark.js`:**

k6 script benchmarking `GET /api/organizations` (secured) vs `GET /api/benchmark/organizations` (baseline) at 1, 10, and 50 VUs (per D-05, D-06).

Structure using k6 scenarios (per RESEARCH pattern 1):
- 6 scenarios total: `secured_1vu`, `baseline_1vu`, `secured_10vu`, `baseline_10vu`, `secured_50vu`, `baseline_50vu`
- Each scenario uses `executor: 'constant-vus'`, `duration: '30s'`
- Stagger start times with `startTime` offsets: 1VU at 0s, 10VU at 35s, 50VU at 70s (5s gap between rounds)
- Tag each scenario with `tags: { endpoint: 'secured' }` or `tags: { endpoint: 'baseline' }`
- Use `exec: 'securedList'` and `exec: 'baselineList'` to route scenarios to different functions

```javascript
import http from 'k6/http';
import { check, sleep } from 'k6';
import { authenticate, authHeaders } from './auth.js';

const BASE_URL = __ENV.BASE_URL || 'http://localhost:8080';

export const options = {
  scenarios: {
    secured_1vu:  { executor: 'constant-vus', vus: 1,  duration: '30s', startTime: '0s',  exec: 'securedList',  tags: { endpoint: 'secured' } },
    baseline_1vu: { executor: 'constant-vus', vus: 1,  duration: '30s', startTime: '0s',  exec: 'baselineList', tags: { endpoint: 'baseline' } },
    secured_10vu:  { executor: 'constant-vus', vus: 10, duration: '30s', startTime: '35s', exec: 'securedList',  tags: { endpoint: 'secured' } },
    baseline_10vu: { executor: 'constant-vus', vus: 10, duration: '30s', startTime: '35s', exec: 'baselineList', tags: { endpoint: 'baseline' } },
    secured_50vu:  { executor: 'constant-vus', vus: 50, duration: '30s', startTime: '70s', exec: 'securedList',  tags: { endpoint: 'secured' } },
    baseline_50vu: { executor: 'constant-vus', vus: 50, duration: '30s', startTime: '70s', exec: 'baselineList', tags: { endpoint: 'baseline' } },
  },
  thresholds: {
    'http_req_duration{endpoint:secured}':  ['p(95)<500'],
    'http_req_duration{endpoint:baseline}': ['p(95)<500'],
  },
};

export function setup() {
  const token = authenticate('admin', 'admin');
  return { token };
}

export function securedList(data) {
  const res = http.get(`${BASE_URL}/api/organizations?page=0&size=20`, {
    headers: authHeaders(data.token),
    tags: { endpoint: 'secured' },
  });
  check(res, { 'secured status 200': (r) => r.status === 200 });
  sleep(1);
}

export function baselineList(data) {
  const res = http.get(`${BASE_URL}/api/benchmark/organizations?page=0&size=20`, {
    headers: authHeaders(data.token),
    tags: { endpoint: 'baseline' },
  });
  check(res, { 'baseline status 200': (r) => r.status === 200 });
  sleep(1);
}
```

Add a `handleSummary(data)` function (per D-03, RESEARCH pitfall 2 — do NOT use `--summary-export`):

```javascript
export function handleSummary(data) {
  // Extract metrics for secured and baseline
  const secP50  = data.metrics['http_req_duration{endpoint:secured}']?.values['p(50)'] ?? 'n/a';
  const secP95  = data.metrics['http_req_duration{endpoint:secured}']?.values['p(95)'] ?? 'n/a';
  const secP99  = data.metrics['http_req_duration{endpoint:secured}']?.values['p(99)'] ?? 'n/a';
  const baseP50 = data.metrics['http_req_duration{endpoint:baseline}']?.values['p(50)'] ?? 'n/a';
  const baseP95 = data.metrics['http_req_duration{endpoint:baseline}']?.values['p(95)'] ?? 'n/a';
  const baseP99 = data.metrics['http_req_duration{endpoint:baseline}']?.values['p(99)'] ?? 'n/a';

  const overheadP95 = typeof secP95 === 'number' && typeof baseP95 === 'number'
    ? ((secP95 - baseP95) / baseP95 * 100).toFixed(1)
    : 'n/a';

  const targetMet = typeof overheadP95 === 'string' && overheadP95 !== 'n/a'
    ? (parseFloat(overheadP95) <= 10 ? 'PASS' : 'FAIL')
    : 'UNKNOWN';

  const md = [
    '# Benchmark: Organization List',
    '',
    `**Date:** ${new Date().toISOString().slice(0, 10)}`,
    `**Target:** p95 overhead < 10% (D-08)`,
    `**Result:** ${targetMet} (${overheadP95}% overhead)`,
    '',
    '| Metric | Secured | Baseline | Overhead |',
    '|--------|---------|----------|----------|',
    `| p50 (ms) | ${fmt(secP50)} | ${fmt(baseP50)} | ${pct(secP50, baseP50)} |`,
    `| p95 (ms) | ${fmt(secP95)} | ${fmt(baseP95)} | ${pct(secP95, baseP95)} |`,
    `| p99 (ms) | ${fmt(secP99)} | ${fmt(baseP99)} | ${pct(secP99, baseP99)} |`,
    '',
    overheadP95 !== 'n/a' && parseFloat(overheadP95) > 10
      ? '> **WARNING:** p95 overhead exceeds 10% target.'
      : '> p95 overhead within 10% target.',
    '',
  ].join('\n');

  return {
    'load-tests/results/org-list-summary.md': md,
    'load-tests/results/org-list-raw.json': JSON.stringify(data, null, 2),
    stdout: textSummary(data, { indent: ' ', enableColors: true }),
  };
}

function fmt(v) { return typeof v === 'number' ? v.toFixed(1) : String(v); }
function pct(a, b) {
  if (typeof a !== 'number' || typeof b !== 'number' || b === 0) return 'n/a';
  return ((a - b) / b * 100).toFixed(1) + '%';
}
```

Import `textSummary` from k6 built-in: `import { textSummary } from 'https://jslib.k6.io/k6-summary/0.1.0/index.js';`

# Performance Benchmarks

k6 load tests comparing the @SecuredEntity pipeline overhead against an unsecured baseline.

## Prerequisites

- **k6** — install via `winget install k6 --source winget` or download from
  https://github.com/grafana/k6/releases
- **Backend running** with dev profile (includes `api-docs` profile group which activates the

## Starting the Backend

```bash
./gradlew bootRun

The dev profile activates by default and includes the `api-docs` profile group. Verify the
benchmark endpoint is available:

```bash
curl -s http://localhost:8080/api/benchmark/organizations -H "Authorization: Bearer <token>" | head -c 200
```

## Running Benchmarks

### Organization List (GET /api/organizations)

```bash
k6 run load-tests/scripts/org-list-benchmark.js
```

### Organization Detail (GET /api/organizations/{id})

```bash
k6 run load-tests/scripts/org-detail-benchmark.js
```

### Custom Configuration

Override the base URL or organization ID:

```bash
k6 run --env BASE_URL=http://localhost:9090 load-tests/scripts/org-list-benchmark.js
k6 run --env ORG_ID=42 load-tests/scripts/org-detail-benchmark.js
```

## Results

After each run, k6 writes results to `load-tests/results/`:

- `org-list-summary.md` — Markdown table with p50/p95/p99 latency comparison
- `org-list-raw.json` — Full k6 metric dump
- `org-detail-summary.md` — Same format for single-entity endpoint
- `org-detail-raw.json` — Full k6 metric dump

### Overhead Target

The acceptable overhead target is **< 10% p95 latency delta** between the secured
`@SecuredEntity` pipeline and the unsecured baseline. The summary report flags if this is
exceeded but does not block builds.

## What Is Being Measured

| Endpoint | Pipeline | Security Checks |
|----------|----------|-----------------|
| `GET /api/organizations` | SecureDataManagerImpl | CRUD check + permission cache + secure serialize |
| `GET /api/benchmark/organizations` | UnconstrainedDataManager | None (same serializer, no permission checks) |

Both endpoints use the same `SecuredEntityJsonAdapter` serializer with the `organization-list`
fetch plan. The only difference is the CRUD permission check and permission cache lookup in
the secured path.

## Concurrency Levels

Each benchmark runs at 1, 10, and 50 virtual users (per the test specification). Each level
runs for 30 seconds with secured and baseline scenarios executing in parallel.
