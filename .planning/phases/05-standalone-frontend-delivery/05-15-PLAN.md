**Component TS (`permission-matrix.component.ts`):**

1. Add imports: `ConfirmationService` from `primeng/api`, `ConfirmDialogModule` from `primeng/confirmdialog`. Add `ConfirmDialogModule` to `imports` array. Add `ConfirmationService` to `providers` array (alongside existing `MessageService`).

2. Inject `ConfirmationService` via `inject(ConfirmationService)`.

3. Add a new state structure for pending changes. Replace the current `pending` Set (which tracks in-flight API calls) with a `pendingChanges` Map that buffers checkbox toggles until Save is clicked:
   ```typescript
   // Key = "target:action", Value = { checked: boolean, targetType: string, target: string, action: string }
   pendingChanges = new Map<string, { checked: boolean; targetType: string; target: string; action: string }>();
   saving = false;
   ```

4. Add a computed property or getter:
   ```typescript
   get hasPendingChanges(): boolean {
     return this.pendingChanges.size > 0;
   }
   ```

5. Add `isPendingChange(target: string, action: string): boolean` method that returns true if the key exists in `pendingChanges`. This is the visual indicator check.

6. Rewrite `togglePermission()` to buffer changes instead of calling the API:
   - Compute the key via `permissionKey(target, action)`.
   - If `pendingChanges` already has this key and the new `checked` value matches the original granted state, REMOVE it from pendingChanges (user toggled back to original).
   - Otherwise, SET `pendingChanges.set(key, { checked, targetType, target, action })`.
   - Do NOT call any API. Do NOT modify `this.granted`.
   - Call `this.cdr.detectChanges()`.

7. Add `isEffectivelyGranted(target: string, action: string): boolean` that reflects the UI checkbox state:
   - If `pendingChanges` has the key, return the pending `checked` value.
   - Otherwise return `this.granted.has(this.permissionKey(target, action))`.
   - Use this in template instead of `isGranted()` for the `[ngModel]` binding.

8. Keep `isGranted()` as-is (checks only `this.granted` map) for internal use.

9. Add `isWildcardEffectivelyGranted(entityCode: string, action: string): boolean` that mirrors `isWildcardGranted` but uses `isEffectivelyGranted`.

10. Add `confirmSave()`:
    ```typescript
    confirmSave(): void {
      this.confirmationService.confirm({
        header: 'Save Permissions',
        message: 'Do you want to save the permission changes?',
        acceptLabel: 'Save',
        rejectLabel: 'Cancel',
        accept: () => this.flushChanges(),
      });
    }
    ```

11. Add `discardChanges()`:
    ```typescript
    discardChanges(): void {
      this.pendingChanges.clear();
      this.cdr.detectChanges();
    }
    ```

12. Add `flushChanges()`:
    - Set `this.saving = true; this.cdr.detectChanges();`.
    - Collect all entries from `pendingChanges` into an array of operations.
    - For each entry: if `checked === true`, build an `ISecPermission` and call `permissionService.create()`. If `checked === false`, get the permission ID from `this.granted.get(key)` and call `permissionService.delete(id)`. If the ID is undefined or < 0, skip (already not persisted).
    - Use `forkJoin` (already imported) to execute all operations in parallel. Each individual observable should be wrapped to catch errors individually (use `pipe(catchError(...))` on each so one failure does not abort others).
    - On each individual success: if it was a create, update `this.granted.set(key, response.body.id)`. If it was a delete, `this.granted.delete(key)`. Remove the key from `pendingChanges`.
    - On each individual error: keep the entry in `pendingChanges` (do not remove). Show a toast via `handleHttpError`.
    - After all complete (forkJoin finalize): set `this.saving = false; this.cdr.detectChanges();`.
    - Import `catchError` and `of` from `rxjs` for the individual error handling.

13. Remove the old `pending` Set entirely. Remove `isPending()` method (replace usages with `saving` flag for disabling during flush).
