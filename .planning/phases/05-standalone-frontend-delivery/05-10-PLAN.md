<task type="auto">
  <name>Task 2: Fix @else if (selectedEntity; as entity) invalid template syntax</name>
  <files>frontend/src/app/pages/admin/security/permission-matrix/permission-matrix.component.html</files>
  <action>
    The attribute panel (second ng-template pTemplate block) contains this three-branch control-flow block:

      @if (!selectedEntity) {
        ...no selection message...
      } @else if (selectedEntity.attributes.length === 0) {
        ...no attributes message...
      } @else if (selectedEntity; as entity) {
        ...attribute table using entity.displayName and entity.attributes...
      }

    The third branch is invalid Angular template syntax — the `; as entity` alias is only legal on the primary `@if`, not on `@else if`. This causes the attribute table to never render.

    Replace the third branch with a plain `@else` block and substitute `selectedEntity!` for every use of `entity` inside that block:

      } @else {
        <h2 class="mb-4 text-base font-semibold">Attribute Permissions: {{ selectedEntity!.displayName }}</h2>

        <p-table [value]="getAttributeRows(selectedEntity!)" [scrollable]="true">
          ...unchanged inner template rows...
        </p-table>
      }

    The two prior branches already guard the null and zero-attribute cases, so `selectedEntity!` is safe here (TypeScript non-null assertion). The inner `ng-template pTemplate="body"` rows are unchanged — they bind `let-row` and do not use `entity`.

    Do not change any other part of the file.
  </action>
  <verify>
    <automated>grep -n "else if (selectedEntity" frontend/src/app/pages/admin/security/permission-matrix/permission-matrix.component.html</automated>
  </verify>
  <done>Zero occurrences of `@else if (selectedEntity` remain. A plain `@else {` branch exists. The build compiles: cd frontend && npx ng build --configuration development 2>&1 | tail -5</done>
