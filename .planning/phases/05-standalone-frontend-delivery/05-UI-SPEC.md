## Spacing Scale

Declared values (multiples of 4). Uses Tailwind utility classes mapped to this scale; PrimeFlex gap classes are the fallback where Tailwind is not applied.

| Token | Value | Tailwind class | Usage |
|-------|-------|---------------|-------|
| xs | 4px | `gap-1`, `p-1` | Icon gaps, badge padding, inline tight spacing |
| sm | 8px | `gap-2`, `p-2` | Button icon-label gap, compact table cell padding |
| md | 16px | `gap-4`, `p-4` | Default element spacing, form field vertical rhythm |
| lg | 24px | `gap-6`, `p-6` | Card internal padding, section padding |
| xl | 32px | `gap-8`, `p-8` | Layout panel padding, major content gaps |
| 2xl | 48px | `gap-12` | Page-level section breaks |
| 3xl | 64px | `gap-16` | Not used in Phase 5 screens |

Exceptions:
- Touch targets for icon-only action buttons: minimum 40px (PrimeNG `p-button-sm` default is sufficient)
- Permission matrix checkbox cells: 44px minimum row height for reliable click area

## Typography

Uses PrimeNG Aura CSS custom properties for color; Tailwind utility classes for size and weight. No custom font is loaded — the Aura preset provides the system font stack.

| Role | Size | Weight | Line Height | Tailwind class |
|------|------|--------|-------------|---------------|
| Body | 14px | 400 (regular) | 1.5 | `text-sm` |
| Label / caption | 12px | 400 (regular) | 1.4 | `text-xs` |
| Heading (page) | 24px | 600 (semibold) | 1.2 | `text-2xl font-semibold` |
| Sub-heading / card title | 16px | 600 (semibold) | 1.3 | `text-base font-semibold` |

Rules:
- Body copy and table cells: 14px weight 400 at 1.5 line-height.
- Page `<h1>` headings: 24px weight 600 at 1.2 line-height (matches `class="text-2xl font-semibold"` in aef-main entity list).
- Muted / secondary text: `text-color-secondary` CSS variable from Aura (do not hardcode a hex).
- No additional type sizes are permitted in Phase 5 screens.

---

## Color

Color is delivered entirely through PrimeNG Aura CSS custom properties. No hardcoded hex values are permitted in component templates except where a semantic CSS variable does not exist.

| Role | CSS Variable / Value | Usage |
|------|---------------------|-------|
| Dominant (60%) | `var(--surface-0)` / `var(--surface-50)` | Page background, layout canvas |
| Secondary (30%) | `var(--surface-100)` / `var(--surface-200)` | Cards (`p-card`), sidebar, topbar, table header rows |
| Accent (10%) | `var(--primary-color)` | Reserved for: primary CTA buttons, active menu item highlight, pagination active page, permission-granted (checked) checkbox cell tint |
| Destructive | `var(--red-500)` / PrimeNG `p-button-danger` severity | Delete buttons, delete confirmation dialogs only |
| Permission denied / absent | No color applied — absent field renders nothing, no red indicator |
| Permission granted indicator | `class="text-green-500"` or `p-checkbox` checked state via Aura — green checkmark, not a custom color |

Accent reserved for EXACTLY these elements:
1. Primary action buttons (`pButton` with `class="primary-button"` or no severity override)
2. Active sidebar menu item background
3. Pagination active page button
4. Checked state of permission matrix `p-checkbox` cells

Dark mode: `app-dark` class on `<html>` flips Aura palette automatically. No manual dark-mode CSS is written.

## Permission Matrix Interaction Contract

Source: CONTEXT.md D-19 through D-27.

### Layout

Two-panel layout using `p-splitter` with vertical orientation:
- Top panel: Entity CRUD matrix table (takes ~60% of available height)
- Bottom panel: Attribute View/Modify matrix for the currently selected entity (takes ~40%)

Bottom panel is empty with a "Select an entity above to view its attributes" message until an entity row is clicked.

### Entity CRUD Matrix

- `p-table` with columns: Entity Name, Create, Read, Update, Delete
- Each data cell contains a single `p-checkbox [binary]="true"`
- Checked state derives from an in-memory `Set<string>` of `"{target}:{action}"` granted keys
- Check = POST `/api/admin/sec/permissions` with `{ authorityName, targetType: "ENTITY", target: entityCode, action: op, effect: "GRANT" }`
- Uncheck = DELETE `/api/admin/sec/permissions/{id}` (id retrieved from current permission list)
- Loading skeleton: show `p-progressSpinner` centered in panel while catalog or permissions are fetching
- No "Save All" button — each checkbox toggle fires immediately (D-23)

### Attribute Matrix

- Appears in bottom panel when entity row is selected
- Columns: Attribute Name, View, Modify
- First row is always the wildcard row (`*`) — displays label "All attributes (*)"
- When wildcard `*` is checked for View or Modify, individual attribute checkboxes for that action become `[disabled]="true"` and visually greyed
- Same immediate-save contract as CRUD matrix

### Visual states

| State | Treatment |
|-------|-----------|
| Checkbox checked | PrimeNG Aura checked state (primary color fill) — no additional CSS |
| Checkbox unchecked | Default unchecked state |
| Checkbox disabled (wildcard active) | `[disabled]="true"` — Aura renders as greyed |
| Row selected in entity table | `[pSelectableRow]` highlight via Aura |
| Loading | `p-progressSpinner` centered in the panel being loaded |
| Save error | `p-toast` with severity `"error"` and the message specified in Copywriting Contract |
| Save success | Silent — no toast (D-27) |

## Protected Entity Field Visibility Contract

Source: CONTEXT.md D-17.

Backend returns `Map<String,Object>` — absent fields are silently missing. TypeScript interfaces use optional types.

| Entity | Sensitive fields | Render rule |
|--------|-----------------|-------------|
| Organization | `budget?: number` | Render only if present in API response — no "N/A" placeholder |
| Employee | `salary?: number` | Render only if present in API response — no "N/A" placeholder |
| All entities | Any absent attribute | Column or field section is omitted entirely from detail and table views |

Table columns for sensitive fields: use `@if (row.budget !== undefined)` or equivalent signal guard — do not show an empty cell.

Form fields for sensitive fields: shown only when the existing entity already carries the value OR when loading a create form (optimistic — the backend enforces actual write permission). If the backend strips the attribute from a load response, the field does not render.
