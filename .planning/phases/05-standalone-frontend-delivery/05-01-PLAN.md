Purpose: The permission matrix UI needs to know which entities and attributes exist (GAP-1) and needs to load only permissions for one role at a time (GAP-2). Without these, the matrix screen cannot function.
Output: Two backend API additions with integration tests.

Create `SecCatalogAdminResource.java` in `com.vn.core.web.rest.admin.security`:
- `@RestController`, `@RequestMapping("/api/admin/sec/catalog")`, `@PreAuthorize("hasAuthority(\"" + AuthoritiesConstants.ADMIN + "\")")`
- Constructor-inject `SecuredEntityCatalog catalog` and `EntityManager entityManager`
- Single `@GetMapping("")` method `getCatalog()` that:
  1. Calls `catalog.entries()` to get all `SecuredEntityEntry` objects
  2. For each entry, enumerates attributes via `entityManager.getMetamodel().entity(entry.entityClass()).getAttributes().stream().map(Attribute::getName).sorted().toList()`
  3. Maps to `SecCatalogEntryDTO` with: `code` = `entry.code()`, `displayName` = `entry.entityClass().getSimpleName()`, `operations` = `entry.operations().stream().map(EntityOp::name).sorted().toList()`, `attributes` = the enumerated attribute names
  4. Returns `ResponseEntity.ok(List<SecCatalogEntryDTO>)`
- Import `jakarta.persistence.metamodel.Attribute` for the attribute name extraction
