```java
private boolean checkAttributePermission(Class<?> entityClass, String attribute, String action) {
    Collection<String> authorities = mergedSecurityService.getCurrentUserAuthorityNames();
    if (authorities.isEmpty()) {
        return false;
    }
    String target = entityClass.getSimpleName().toUpperCase(Locale.ROOT) + "." + attribute.toUpperCase(Locale.ROOT);
    List<SecPermission> perms = secPermissionRepository.findByRolesAndTarget(authorities, TargetType.ATTRIBUTE, target, action);
    // BUG: No rules for this attribute = permissive default (returns true)
    // Should be deny-default for secured catalog entities
    if (perms.isEmpty()) {
        return true;  // <-- MUST change to false
    }
    boolean hasDeny = perms.stream().anyMatch(p -> "DENY".equals(p.getEffect()));
    if (hasDeny) { return false; }
    return perms.stream().anyMatch(p -> "ALLOW".equals(p.getEffect()));
}
```

    **Why:** The permission matrix UI only stores GRANT records (effect=ALLOW). Unchecking a permission
    deletes the GRANT row. AttributePermissionEvaluatorImpl currently returns true when perms list is
    empty (line 52-53: `if (perms.isEmpty()) { return true; }`). This means unchecked attributes are
    treated as allowed, making the entire attribute permission matrix ineffective.

    The Phase 03-02 decision noted "permissive-default: empty permission list returns true (no rules = allowed)".
    That was correct for unregistered entities (no rules = no restrictions). But the capability endpoint
    only queries attributes of entities IN the secured catalog (SecuredEntityCapabilityService iterates
    securedEntityCatalog.entries()). For these explicitly registered entities, "no GRANT record" must
    mean "denied" -- otherwise the matrix has no way to deny attribute access.

    **Fix in AttributePermissionEvaluatorImpl.java:**

    1. Change line 52-53 from:
       ```java
       // No rules for this attribute = permissive default
       if (perms.isEmpty()) {
           return true;
       }
       ```
       To:
       ```java
       // No GRANT record = denied. The permission matrix stores only GRANT records,
       // so empty results means no grant was given for this attribute.
       if (perms.isEmpty()) {
           return false;
       }
       ```

    2. Update the Javadoc at class level: change "default to permissive" to "default to restrictive"
       and update the description to say "if no permission records exist for the given attribute,
       access is denied."

    2. In `loadCapability()`, after extracting canUpdate, also build a field visibility map from `capability.attributes`:
       ```typescript
       private loadCapability(): void {
         this.securedEntityCapabilityService
           .getEntityCapability('organization') // or 'department' or 'employee'
           .pipe(take(1))
           .subscribe({
             next: capability => {
               this.canUpdate.set(capability?.canUpdate ?? false);
               if (capability?.attributes) {
                 const vis: Record<string, boolean> = {};
                 for (const attr of capability.attributes) {
                   vis[attr.name] = attr.canView;
                 }
                 this.fieldVisibility.set(vis);
               }
               this.capabilityLoaded.set(true);
             },
             error: () => {
               this.canUpdate.set(false);
               this.capabilityLoaded.set(true);
             },
           });
       }
       ```
    3. Add a helper method using CORRECTED logic (addresses checker warning about empty-array blanking):
       ```typescript
       canViewField(fieldName: string): boolean {
         // Before capability loads, show all fields (they will be gated once capability arrives)
         if (!this.capabilityLoaded()) {
           return true;
         }
         // After capability loads, check the visibility map
         const vis = this.fieldVisibility();
         // If field is explicitly set to false, hide it
         // If field is not in the map at all (not a secured attribute), show it
         return vis[fieldName] !== false;
       }
       ```
       **IMPORTANT:** Do NOT use `Object.keys(vis).length === 0` as a branching condition.
       The correct approach is: check `capabilityLoaded()` first. If not loaded, return true
       (show fields optimistically). If loaded, use `vis[fieldName] !== false` which correctly
       handles both: (a) empty attributes array (all fields visible -- no restrictions), and
       (b) fields not in the map (visible by default). This decouples "not yet loaded" from
       "loaded with no attribute restrictions".
