    **TypeScript changes (all three detail .component.ts files):**

    1. Add a new signal: `fieldVisibility = signal<Record<string, boolean>>({});`
    2. In `loadCapability()`, after extracting canUpdate, also build a field visibility map from `capability.attributes`:
       ```typescript
       private loadCapability(): void {
         this.securedEntityCapabilityService
           .getEntityCapability('organization') // or 'department' or 'employee'
           .pipe(take(1))
           .subscribe({
             next: capability => {
               this.canUpdate.set(capability?.canUpdate ?? false);
               if (capability?.attributes) {
                 const vis: Record<string, boolean> = {};
                 for (const attr of capability.attributes) {
                   vis[attr.name] = attr.canView;
                 }
                 this.fieldVisibility.set(vis);
               }
               this.capabilityLoaded.set(true);
             },
             error: () => {
               this.canUpdate.set(false);
               this.capabilityLoaded.set(true);
             },
           });
       }
       ```
    3. Add a helper method:
       ```typescript
       canViewField(fieldName: string): boolean {
         const vis = this.fieldVisibility();
         // If no visibility info loaded yet (empty object), hide fields until capability loads
         if (Object.keys(vis).length === 0) {
           return !this.capabilityLoaded();
         }
         // If field not in map (not a secured attribute), show it
         return vis[fieldName] !== false;
       }
       ```
       The logic: before capability loads, show fields (they will be replaced when capability arrives). After capability loads, if a field is explicitly `false` in the map, hide it. If the field is not in the map at all (e.g. 'id' which may not be a secured attribute), show it.

    **Template changes:**

    **organization-detail.component.html** -- wrap each data field (except ID which is structural) with `@if (canViewField('fieldName'))`:
    - Code field: `@if (canViewField('code')) { <div>...</div> }`
    - Name field: `@if (canViewField('name')) { <div>...</div> }`
    - Owner Login field: `@if (canViewField('ownerLogin')) { <div>...</div> }`
    - Budget field: already has `@if (org.budget !== undefined)` -- change to `@if (canViewField('budget') && org.budget !== undefined)`

    **department-detail.component.html** -- same pattern:
    - Code field: `@if (canViewField('code')) { ... }`
    - Name field: `@if (canViewField('name')) { ... }`
    - Organization field: `@if (canViewField('organization') && dept.organization) { ... }`
    - Cost Center field: `@if (canViewField('costCenter') && dept.costCenter !== undefined) { ... }`

    **employee-detail.component.html** -- same pattern:
    - Employee Number field: `@if (canViewField('employeeNumber')) { ... }`
    - First Name field: `@if (canViewField('firstName')) { ... }`
    - Last Name field: `@if (canViewField('lastName')) { ... }`
    - Department field: `@if (canViewField('department') && emp.department) { ... }`
    - Email field: `@if (canViewField('email') && emp.email !== undefined) { ... }`
    - Salary field: `@if (canViewField('salary') && emp.salary !== undefined) { ... }`

    ID fields stay ungated (they are structural identifiers, not sensitive data).
