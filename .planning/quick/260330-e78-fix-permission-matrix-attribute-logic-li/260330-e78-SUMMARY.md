## What Was Built

### Modify-Implies-View

`isViewImpliedByModify(target, entityCode)` returns true when EDIT is effectively granted for the
given attribute (either explicitly on `entityCode.attr` or via the wildcard `entityCode.*:EDIT`).
This mirrors Jmix behaviour where modify access inherently grants view access.

In the template, the attribute VIEW checkbox now:
- Shows as checked when either `isAttributeEffectivelyGranted(..., 'VIEW', ...)` OR `isViewImpliedByModify(...)` is true
- Is disabled when `isViewImpliedByModify(...)` is true (cannot be unchecked)

### Entity Wildcard Row

`catalogEntriesWithWildcard` prepends a synthetic `{ code: '*', displayName: 'All entities (*)', ... }` entry to the entity table. The entity table now uses this getter instead of `catalogEntries`.

`isEntityWildcardEffectivelyGranted(action)` checks `*:action` in pending or granted state.
`isEntityEffectivelyGranted(entityCode, action)` returns true for explicit grant OR when wildcard is active.

Individual entity checkboxes are disabled when wildcard is active for that action (entity is not the wildcard row itself). The wildcard row is styled with `font-semibold` and `surface-50` background and is not clickable for attribute panel selection.

Wildcard entity permissions save with `target='*'` and `targetType='ENTITY'`, consistent with how attribute wildcards use `entityCode.*`.

### i18n Keys Added

- `security.permissionMatrix.entity.wildcard` = "All entities (*)" (en)
- `security.permissionMatrix.entity.wildcard` = "Tất cả thực thể (*)" (vi)
