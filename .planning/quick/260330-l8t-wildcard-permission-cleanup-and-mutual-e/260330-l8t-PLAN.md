# Quick Task: Wildcard Permission Cleanup and Mutual Exclusivity in createPermission

## Objective

Fix two issues in `SecPermissionAdminResource.createPermission`:

1. **Missing `normalizeOutgoing` on duplicate-handling path** — when a matching permission already exists the method returns `ResponseEntity.ok(secPermissionMapper.toDto(canonicalPermission))` without calling `secPermissionUiContractService.normalizeOutgoing(...)`. This means the response returns the raw stored format (e.g. `target: "ORGANIZATION"`, `effect: "ALLOW"`) instead of the UI contract format (`target: "organization"`, `effect: "GRANT"`), inconsistent with every GET endpoint.

2. **Missing IT coverage for entity wildcard (`*`) round-trip** — no existing test sends `target: "*"` through the create-and-read path, so the pass-through normalization for the wildcard is not regression-tested.

## Tasks

### Task 1 — Fix normalizeOutgoing on duplicate path in createPermission

In `SecPermissionAdminResource.createPermission`, the duplicate-handling branch returns the canonical permission DTO without normalizing it back to UI-contract format.

Change:
```java
return ResponseEntity.ok(secPermissionMapper.toDto(canonicalPermission));
```
To:
```java
return ResponseEntity.ok(secPermissionUiContractService.normalizeOutgoing(secPermissionMapper.toDto(canonicalPermission)));
```

Also update the `createPermission_matrixGrantReturnsExistingRowInsteadOfCreatingDuplicate` IT test to expect the normalized UI-contract values (`target: "organization"`, `effect: "GRANT"`) instead of the raw stored values.

### Task 2 — Add IT test for entity wildcard `*` create and roundtrip

Add test `createPermission_entityWildcardGrantStoresAsWildcardAndRoundTripsAsGrant` to `SecPermissionAdminResourceIT`:

- POST `{ targetType: "ENTITY", target: "*", action: "READ", effect: "GRANT", authorityName: "ROLE_PROOF_NONE" }`
- Assert persisted entity has `target = "*"` and `effect = "ALLOW"`
- Assert GET by authority returns `target: "*"` and `effect: "GRANT"`
- Assert GET by id returns `target: "*"` and `effect: "GRANT"`
